Reflection

Data Processing and Transformation

The initial phase of my project was important, laying the groundwork for all subsequent analyses. The tree data had a lot of missing values, and poorly labeled ones so I had to clean it. One task that stood out was refining the street names. The complexity of this process was underscored by the iterative Python scripts I coded. These scripts not only helped in standardizing the street names but also played an instrumental role in other preprocessing tasks, such as handling missing values and segregating tree species. Another instance I used Python was to split the species name into the genus and species. This way I could use the genus instead of the species in my analysis, because the species column had verbal misspellings and other issues. A third instance was assigning a date to the trees that only had a year. I chose to use the 1st of January of that year, so I could process the data by year at least on Tableau. There were instances where certain data transformation strategies were contemplated, such as outlier treatments and advanced categorizations. However, some of these did not transition to the final workbook. The primary reason behind these exclusions was my commitment to preserving the integrity and authenticity of the original data, ensuring that my analyses remained true to Lawrence's tree landscape, and the data set I was given. With further analysis and more experience this project could be revisited, possibly providing richer insights.

Data Exploration

My strategy to analyze and visualize this data was simple. Begin by exploring any changes over time, geospatial data, and species data. This way I have an understanding of some key dimensions. Then after taking this understanding, attempting to find relationships between all these dimensions. However, even to get to the last phase my initial visualizations were filled with challenges. One prominent hurdle was the geospatial clustering based on latitude and longitude. While the idea was to visualize tree densities in distinct neighborhoods, achieving this proved more intricate than anticipated due to software constraints. This limitation led us to a crossroads: should I pursue a workaround or pivot to a different visualization approach? Another issue arose when I attempted to understand species diversity. The sheer variety of tree species planted in Lawrence posed a categorization challenge. Deciding on how to effectively group and visualize them without oversimplifying or complicating the narrative required multiple iterations. Furthermore, the nested pie chart, intended to depict the dominance of certain species, took several tweaks before it effectively conveyed the desired insights. I also grappled with the balance between providing comprehensive data and avoiding information overload for the viewer. Lastly, while diving deep into street-level data, inconsistencies in street naming conventions surfaced. This prompted a revisit to the data processing phase, underlining the iterative nature of data exploration. Once these issues were taken care of I was able to ask more interesting questions about the data, which you can see in the Tableau story. Those visualizations were complex and required a lot more thinking and planning, but I was able to gather some interesting insight from them as you saw above.
